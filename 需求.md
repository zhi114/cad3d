# 功能概要

我想实现一套根据dxf文件绘制3d模型的系统
整个流程为用户通过浏览器上传dxf文件，然后服务器保存该文件并进行解析，之后在浏览器上得到解析数据并生成对应的3d模型

实现分为前端和后端

# 前端技术说明

1.前端负责dxf文件的上传和3d模型的绘制2.前端技术栈：react+typescipt+vite+three.js

# 后端技术说明

1.后端负责dxf文件的解析和向前端提供dxf文件信息的api服务2.后端技术栈：python+ezdxf

# 解析规则

用户的dxf文件内容为2d平面图。解析时只需要保留墙/门/窗/天线这几个图层
dxf图层的命名规则如下：
墙：WALL
门：DOOR
窗：WINDOW
天线：ANTENNA
包含以上名称的图层才会被前端3d化

# 3d化规则

墙的默认高度为3m
门的高度默认为1.5m
窗户的高度默认为1m，离地高度默认为1m
